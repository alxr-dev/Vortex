# Vortex

Проект представляет собой набор скриптов на Python для автоматизированного поиска каналов и мониторинга контента в мессенджере Telegram.

Программное решение разделено на два изолированных модуля, работающих с текстовыми базами данных (`.csv` и `.txt`): скрипт для парсинга (Grabber) и скрипт для отслеживания триггеров и отправки уведомлений (Monitor).

## Функциональные особенности

* **Работа по протоколу MTProto:** Взаимодействие с Telegram API реализовано через библиотеку `Telethon` асинхронно (`asyncio`), что позволяет перехватывать события и парсить данные быстрее, чем через стандартный HTTP Bot API.
* **Событийный мониторинг сообщений:** Скрипт отслеживания не делает периодических запросов для проверки обновлений, а слушает поток входящих событий Telegram в реальном времени.
* **Уведомления через Telegram-бота:** Оповещения о найденных триггерах отправляются через Bot API (библиотека `aiogram 3.x`), что изолирует рабочий аккаунт от процесса доставки уведомлений.
* **Два режима поиска каналов:** * `contacts` — аналог встроенной поисковой строки Telegram. Ищет публичные каналы и группы по ключевым словам по всей глобальной базе мессенджера (выдача ограничена лимитами Telegram). Присутствие в целевых каналах не требуется.
* `global` — сквозной глобальный поиск по постам и непубличным каналам (для работы требуется аккаунт Telegram Premium).


* **Файловая конфигурация:** Списки целевых каналов и отслеживаемые фразы вынесены в отдельные файлы в директории `data/`. Для применения изменений в списках ключевых слов требуется перезапуск скриптов.

## Технологический стек

* Python 3.12+
* Telethon (MTProto API)
* aiogram 3.x (Bot API)
* csv
* asyncio

## Настройка и развертывание

1. Клонируйте репозиторий и перейдите в папку проекта:

```bash
git clone https://github.com/alxr-dev/Vortex.git
cd Vortex

```

2. Создайте и активируйте виртуальное окружение:

```bash
python -m venv .venv
# Для Windows:
.venv\Scripts\activate
# Для Linux/Mac:
source .venv/bin/activate

```

3. Установите необходимые пакеты:

```bash
pip install -r requirements.txt

```

## Переменные окружения

Перед запуском необходимо настроить следующие переменные в вашей системе (`set` - Windows, `export` - Linux):

```bash
# Данные авторизации аккаунта (получать на my.telegram.org)
TELETHON_API_ID="your_api_id"
TELETHON_API_HASH="your_api_hash"

# Данные бота уведомлений
BOT_TOKEN="your_bot_token"       # Токен от @BotFather
USER_CHAT_ID="your_chat_id"     # ID пользователя или ID группы, куда бот будет слать алерты

```

## Инструкция по работе со скриптами

Процесс работы с системой делится на три этапа: поиск источников, подготовка списков и запуск отслеживания.

### Этап 1: Поиск целевых источников (Grabber)

Заполните файл `data/keywords.txt` ключевыми словами или словосочетаниями (по одному на строку). Запустите скрипт поиска из корня проекта:

```bash
python -m grabber.telegram_grabber --mode both --search-mode contacts --limit 50

```

Скрипт найдет публичные ресурсы по вашей базе запросов и сохранит их параметры (`id`, `title`, `username`, `participants_count`, `description`) в файлы `data/chats.csv` и `data/channels.csv`.

### Этап 2: Фильтрация и подготовка списков

1. Изучите сгенерированные файлы `chats.csv` и `channels.csv`, удалите нерелевантные строки.
2. Перенесите отфильтрованные данные в файлы `data/monitor_chats.csv` и `data/monitor_channels.csv` — именно их будет использовать модуль мониторинга.
3. Внесите целевые слова и фразы для отслеживания в файл `data/monitor_keywords.txt`.

### Этап 3: Запуск мониторинга (Monitor)

Запустите скрипт отслеживания:

```bash
python -m monitor.monitor_bot

```

При первом запуске скрипт потребует ввести код подтверждения из Telegram для инициализации сессии. После этого программа перейдет в режим постоянного ожидания. При появлении ключевого слова или словосочетания в отслеживаемых ресурсах, бот мгновенно отправит сообщение со структурированным текстом и ссылкой на пост в личные сообщения пользователя или указанный чат (в зависимости от переданного `USER_CHAT_ID`).

## Важное замечание по безопасности

При первой авторизации в корне проекта создается файл сессии Telethon (по умолчанию `session.session`).

**Ни в коем случае не публикуйте этот файл в открытом доступе. Данный файл содержит полные права доступа к вашему Telegram-аккаунту. Его компрометация приведет к потере контроля над аккаунтом.**
